2. REAL LINK HANDLING
=====================

When a user saves a link, I want the system to do more than just store the URL.

Required behavior:

* save the original URL
* try to fetch or extract useful metadata:

  * page title
  * description
  * domain
  * main text if realistically possible
* save this extracted content into the memory item
* make the link searchable by its extracted content and not only by the raw URL
* show the link item in a rich way in the UI

If direct scraping is limited, implement the safest realistic fallback:

* title + description + domain + optional fetched preview content
